## 状态定义

| 状态 | 含义 |
| --- | --- |
| `open` | 已有可重复事实，仍需处理。 |
| `fact-gap` | 事实不足，不能用猜测推进。 |
| `deferred` | 已记录，但当前阶段不处理。 |
| `fixed` | 已有实现和验证证据。 |

## 记录规则

- 事实证据应写入对应 `facts/` 或专题事实文档，再从本台账引用。
- 不能把测试缓存、临时配置或手工修改 `temp` 文件作为 production 默认策略依据。
- 修改方案必须说明是否会阻塞 normal build，以及与 Editor / AssetDB / `@cocos/lib-programming` 行为的关系。
